# Design Tokens

디자인 토큰은 컴포넌트의 시각적 속성을 체계적으로 관리하는 기반입니다.
이 시스템은 **3계층 구조**로 토큰을 설계했습니다.

## 토큰 아키텍처

```
Primitive → Semantic → Component
```

### 왜 3계층인가?

- **Primitive만 사용하면**: `gray-500`이 텍스트인지 배경인지 알 수 없음
- **Semantic을 거치면**: 다크모드에서 `--color-text-secondary`만 바꾸면 모든 컴포넌트가 자동 변환
- **Component에서 직접 값을 쓰면**: 테마 변경 시 모든 컴포넌트를 하나씩 수정해야 함

## Color

### Primitive Colors

원시 색상은 gray, blue, red, green, amber 5개 팔레트로 구성됩니다.
각 팔레트는 50~900 스케일을 가집니다.

> **"Colors" 스토리**에서 전체 팔레트를 확인할 수 있습니다.

## 다크모드

CSS 변수 기반으로 `data-theme` 속성 전환만으로 다크모드를 지원합니다.

```css
/* light (기본) */
:root {
  --color-bg-primary: #ffffff;
  --color-text-primary: #111827;
}

/* dark */
[data-theme='dark'] {
  --color-bg-primary: #111827;
  --color-text-primary: #f9fafb;
}
```

컴포넌트 코드 변경 없이 자동으로 테마가 전환됩니다.
